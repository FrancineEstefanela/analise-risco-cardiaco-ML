  • thalach: Frequência cardíaca máxima atingida.
  • exang: Angina induzida pelo exercício (0: não, 1: sim).
  • oldpeak: Depressão do segmento ST.
  • slope: Inclinação do segmento ST no pico do exercício.
  • ca: Número de vasos principais colorido por fluoroscopia.
  • thal: Teste de esforço cardíaco (3: normal, 6: defeito fixo, 7: defeito reversível).
  • num: Diagnóstico de doença cardíaca (0: não há, 1-4: há doença).

Nesta etapa, vamos preparar os dados para os modelos de machine learning, convertendo as variáveis e normalizando os valores para análise mais eficiente. Após vamos treinar os modelos de Random Forest e Regressão Logística com os dados de treino, vamos comparar os resultados utilizando o classification report que apresenta as métricas de avaliação, como precisão, recall, f1-score e acurácia.

In [14]:
# Separação das variáveis
X = df_uci.drop('num', axis=1)
y = df_uci['num']

# Padronização dos dados
X = StandardScaler().fit_transform(X)

# Divisão dos dados em treino e teste (80% treino, 20% teste)
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.28, random_state=42)

# Definindo as métricas a serem comparadas
metrics = ['Precision', 'Recall', 'F1-Score']
rf_scores = [
    0.79,  # Precision para classe 0 (Random Forest)
    0.84,  # Recall para classe 0 (Random Forest)
    0.81   # F1-Score para classe 0 (Random Forest)
]
log_reg_scores = [
    0.88,  # Precision para classe 0 (Logistic Regression)
    0.86,  # Recall para classe 0 (Logistic Regression)
    0.87   # F1-Score para classe 0 (Logistic Regression)
]

# Criando o gráfico

width = 0.3  # Largura das barras

# Adicionando rótulos e título
ax.set_xlabel('Métricas')

ax.set_xticks(x)
ax.set_xticklabels(metrics)
ax.legend()

# Exibindo os valores nas barras
def add_values(bars):
    for bar in bars:
        height = bar.get_height()
        ax.annotate(f'{height:.2f}', 
                    xy=(bar.get_x() + bar.get_width() / 2, height),
                    xytext=(0, 3),  # Deslocamento para cima
                    textcoords="offset points",
                    ha='center', va='bottom')

# Adicionando os valores às barras
add_values(bars_rf)
add_values(bars_log_reg)

# Exibindo o gráfico
plt.show()

Conclusão: A Regressão Logística apresentou melhor desempenho geral, especialmente em termos de acurácia, recall e F1-score, sendo mais equilibrada. Embora o Random Forest tenha sido bom em precisão para a classe 1, a Regressão Logística é mais indicada para esse problema.

  • Acurácia: A Regressão Logística obteve melhor desempenho (87% vs. 80%).
  • Precisão: O Random Forest teve precisão ligeiramente melhor para a classe 1 (com doença), enquanto a Regressão Logística foi melhor para a classe 0 (sem doença).
  • Recall: A Regressão Logística foi mais equilibrada, com recall superior tanto para a classe 0 quanto para a classe 1.
  • F1-Score: A Regressão Logística também teve um F1-score mais equilibrado.

Conclusão Final¶

Na análise dos dados, observei que a variável alvo apresenta uma leve predominância de pacientes sem doença cardíaca (164 casos) em comparação aos pacientes com a doença (139 casos). Além disso, em relação à distribuição por sexo, a maior parte dos pacientes são homens (cerca de 68%), enquanto as mulheres representam cerca de um terço do grupo.

Quanto à idade e aos níveis de colesterol, percebi um padrão interessante: a partir dos 40 anos, os níveis de colesterol começam a subir, com um aumento mais acentuado a partir dos 50 anos. Isso destaca a importância de monitoramento regular da saúde, especialmente para homens na meia-idade, a fim de identificar e prevenir doenças cardíacas.

Em relação aos modelos de aprendizado de máquina, a matriz de confusão foi útil para avaliar o desempenho dos modelos. Ambos, Random Forest e Regressão Logística, apresentaram boas taxas de acerto. No entanto, a Regressão Logística se destacou ligeiramente na previsão das classes positivas, cometendo menos erros. Já o Random Forest teve um desempenho mais equilibrado, mas com um número um pouco maior de erros nas classes positivas, indicando que, embora eficaz, ele pode ser mais sensível a esse tipo de erro.
